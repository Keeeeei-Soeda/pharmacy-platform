## プロジェクト概要

### サービス概要

薬剤師と薬局をマッチングする人材紹介プラットフォーム。求職者が直接応募し、短期雇用契約でトライアル勤務を行うことで、双方が納得した上での本採用につなげるサービス。

### ビジネスモデル

- 薬剤師が求人に直接応募
- 短期雇用契約でトライアル勤務
- 双方納得した上で本採用へ

### 主な訴求ポイント

#### 薬局側

- ✅ 実際の働きを見てから採用できる
- ✅ 採用の失敗リスクを大幅に削減
- ✅ 採用コストの削減

#### 薬剤師側

- ✅ 実際に働いてから転職を判断できる
- ✅ 給料を得ながら職場を吟味
- ✅ 複数の店舗を体験可能

## 技術スタック

### フロントエンド

- **フレームワーク**: Next.js 15.5.4 (App Router)
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS
- **UI コンポーネント**: Lucide React (アイコン)
- **状態管理**: React Hooks (useState, useEffect)
- **HTTP クライアント**: Fetch API

### バックエンド

- **ランタイム**: Node.js v22.18.0
- **フレームワーク**: Express.js
- **言語**: JavaScript
- **認証**: JWT (JSON Web Tokens)
- **プロセス管理**: Nodemon

### データベース

- **DBMS**: PostgreSQL
- **ORM**: Prisma 6.16.2
- **マイグレーション**: Prisma Migrate

### 開発環境

- **バージョン管理**: Git
- **パッケージマネージャー**: npm
- **開発サーバー**:
  - フロントエンド: http://localhost:3000 (Turbopack)
  - バックエンド: http://localhost:3001

### 今後導入予定

- **PDF 生成**: PDFKit または jsPDF（請求書・源泉徴収票）
- **カレンダー UI**: react-big-calendar または react-calendar
- **メール送信**: Nodemailer（SMTP）
- **ファイルストレージ**: AWS S3 または同等（薬剤師免許アップロード用）

## 既知の課題

### 技術的課題

#### 1. Prisma Client の非同期性 ⚠️

- **問題**: スキーマ変更後、サーバー再起動でも古いクライアントが読み込まれることがある
- **影響**: `status` フィールドなどの新規フィールドが認識されない
- **解決策**: 完全プロセス kill + Prisma Client 再生成
- **恒久対策**: CI/CD パイプラインでの自動再生成

#### 2. フロントエンドポート競合 ℹ️

- **問題**: Port 3000 が別プロセスで使用中
- **現状**: Port 3002 で起動
- **影響**: なし（正常動作）
- **解決策**: 不要なプロセスの特定・終了

#### 3. Next.js Turbopack 警告 ℹ️

- **問題**: ワークスペースルートの推論警告
- **現状**: 動作に影響なし
- **解決策**: `next.config.ts` に `turbopack.root` 設定追加
