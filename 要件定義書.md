# 薬剤師マッチングプラットフォーム要件定義書

## 1. システム概要

### 1.1 サービス概要
薬剤師と薬局をマッチングする人材紹介プラットフォーム。求職者が直接応募し、短期雇用契約でトライアル勤務を行うことで、双方が納得した上での本採用につなげるサービス。

### 1.2 ユーザー定義
- **薬局**：薬剤師を募集している事業者
- **薬剤師（求職者）**：薬剤師として働きたい人

### 1.3 ビジネスモデル
求職者が求人情報に直接応募し、短期雇用契約を結んでトライアル勤務を実施

## 2. 訴求ポイント

### 2.1 薬局側のメリット
- 実際の働きを見てから採用できる
- 採用の失敗リスクを回避できる
- 採用コストを大幅に削減できる

### 2.2 薬剤師側のメリット
- 実際に働いてから転職を判断できる
- 給料を得ながら転職先を吟味できる
- 複数の店舗を見て回ることができる

## 3. 機能要件

### 3.1 求人管理機能
- **求人掲載機能**
  - 薬局による求人情報の入力・掲載
  - 労働条件の詳細掲載
  - 入力完了後の自動掲載機能
- **求人管理機能**
  - 案件の掲載開始・停止
  - 掲載状況の確認

### 3.2 ユーザー管理機能
- **薬剤師アカウント機能**
  - アカウント作成機能
  - 薬剤師免許・保険薬剤師登録票のアップロード機能
  - プロフィール管理
- **薬局アカウント機能**
  - アカウント作成・管理
  - 薬局情報の管理

### 3.3 応募・マッチング機能
- **応募機能**
  - 求職者からの応募機能
  - 利用規約同意チェック機能
  - 応募時の薬局への通知機能
- **採用可否機能**
  - 薬局から求職者への応募承認・拒否通知
  - 応募薬剤師のプロフィール確認
- **アクセス制限**
  - アカウント作成済みユーザーのみ応募・閲覧可能

### 3.4 スケジュール・勤怠管理機能
- **スケジュール管理**
  - 求人薬局のスケジュール管理
- **勤怠管理**
  - 出勤・退勤時間の記録機能（LINE連携）
  - 薬剤師が入力、薬局が管理する仕組み
- **給与計算**
  - 勤務時間から時給を掛けた給与計算
  - 請求額の自動計算

### 3.5 帳票・請求機能
- **請求書作成**
  - 勤務時間に基づく請求書の自動作成
  - PDF化機能
  - 請求書送信機能
- **源泉徴収票**
  - 求職者アカウント内での発行申込機能

### 3.6 ダッシュボード機能
- **薬局側ダッシュボード**
  - 案件掲載管理
  - 応募状況確認
  - 応募者プロフィール確認
  - 採用可否決定
  - 通知機能
- **薬剤師側ダッシュボード**
  - 応募管理
  - 応募フォーム入力
  - 応募案件管理（応募上限管理）
  - 通知機能

### 3.7 その他機能
- **問い合わせページ**
- **訴求ポイントまとめページ**（薬局・薬剤師別）
- **通知機能**（応募、承認・拒否等）

## 4. 技術要件

### 4.1 システム構成
- **WEBアプリケーション**：メイン機能
- **LINE公式アカウント**：勤怠管理機能
- **システム連携**：WEBアプリ ↔ LINE

### 4.2 主要技術
- フロントエンド：Webアプリケーション
- バックエンド：API開発
- データベース：ユーザー情報、求人情報、勤怠情報等の管理
- 外部連携：LINE Messaging API

## 5. 開発・運用要件

### 5.1 開発項目
- LINE構築
- WEBアプリ構築
- LINE + WEBアプリ連携
- 要件定義
- デバッグ・動作確認

### 5.2 概算予算
- **総額**：最低100万円程度
- **内訳**：
  - LINE構築：20万円〜
  - WEBアプリ構築：50万円〜（機能が多いため一般的なものより高め）
  - LINE + WEBアプリ連携：30万円〜
  - 要件定義、デバッグ、動作確認：10〜30万円

## 6. 補足事項

### 6.1 薬局側の作業負荷軽減
- 薬局側の入力作業を最小限に抑制
- 事前準備されたテンプレートに必要事項を入力するのみ
- テンプレートとHP掲載の連携機能

- 決済機能の必要性
